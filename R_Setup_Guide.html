Setting Up R and VS Code

A complete guide for running R scripts and Shiny apps on Windows using Visual Studio Code.

1. Install R

R is the language interpreter. VS Code is just an editor — you still need R itself installed.

1Go to https://cran.r-project.org/

2Click Download R for Windows.

3Click base, then Download R-4.x.x for Windows.

4Run the installer with all default options.

Note the version. After install, R lives in C:\Program Files\R\R-4.x.x\ — the exact 4.x.x depends on the version you downloaded. You'll need this path later.

2. Install Visual Studio Code

1Go to https://code.visualstudio.com

2Download the Windows installer and run it.

3Tick "Add to PATH" and "Register Code as an editor for supported file types" when prompted.

3. Install the R Extension

1Open VS Code.

2Click the Extensions icon on the left sidebar (or press Ctrl+Shift+X).

3Search for R.

4Install the extension by REditorSupport (publisher ID: REditorSupport.r).

4. Install Required R Packages

The R extension and the Shiny app both depend on a handful of packages. Install them once, up front.

1In VS Code, open the integrated terminal: Ctrl+` (the backtick key, top-left of the keyboard).

2Open the Command Palette: Ctrl+Shift+P, type R: Create R Terminal, press Enter.

3You should see a > prompt — this is the R console.

4Paste this command and press Enter:

install.packages(c("jsonlite", "rlang", "languageserver", "httpgd", "shiny", "DT", "ggplot2", "numDeriv"))

5If asked to choose a CRAN mirror, pick 0-Cloud [https].

6Wait 1-3 minutes for installation to finish.

PackagePurpose
jsonlite, rlangRequired by VS Code's R session watcher.
languageserverProvides autocomplete and code diagnostics inside VS Code.
httpgdRenders plots inside a VS Code panel.
shinyFramework for building interactive web apps.
DTInteractive HTML data tables.
ggplot2Plotting and visualization.
numDerivNumerical differentiation utilities.

5. Configure VS Code Settings

Tell VS Code where R lives and how to behave with it.

1Press Ctrl+, in VS Code to open Settings.

2In the search bar at the top, type R.

3Configure the following three settings:

SettingWhat to doWhy
R > Rterm: Windows Paste the full path to R.exe, e.g.
C:\Program Files\R\R-4.4.1\bin\x64\R.exe		 Tells VS Code where to find R.
R > Plot: Use Httpgd Tick the checkbox. Renders plots inline in a VS Code panel.
R > Bracketed Paste Tick the checkbox. Sends multi-line code as one block, avoiding parse errors.
Find your exact R path: open File Explorer and navigate to C:\Program Files\R\. You'll see a folder like R-4.4.1. Inside, go to bin\x64\ and copy the address bar — append \R.exe.

6. Run Your First R Script

1Open any .R file in VS Code (e.g. GaussJacobi_App.R).

2Place the cursor on a line of code.

3Press Ctrl+Enter to send that line to the R terminal.

4Select multiple lines and press Ctrl+Enter to send a block.

7. Run a Shiny App

1Make sure your R terminal is open (the one with the > prompt).

2Run:

shiny::runApp("GaussJacobi_App.R", launch.browser = TRUE)

3The app opens in your default browser at an address like http://127.0.0.1:xxxx.

4To stop the app, click in the R terminal and press Esc or Ctrl+C.

Shortcut: from any terminal (PowerShell or R) you can also run: 
R -e "shiny::runApp('GaussJacobi_App.R', launch.browser=TRUE)"

8. Troubleshooting

Error: "unexpected symbol in (Set-ExecutionPolicy ..."

You pasted a PowerShell command into the R terminal. R doesn't understand PowerShell syntax.

  • R prompt looks like: >
  • PowerShell prompt looks like: PS C:\Users\You>

Make sure you're in the right terminal before pasting commands. The error itself is harmless — just ignore it.

"VSCode R Session Watcher requires jsonlite, rlang"

You skipped Section 4. In an R terminal, run:

install.packages(c("jsonlite", "rlang"))

Plots don't appear

Confirm httpgd is installed and R > Plot: Use Httpgd is ticked. Restart VS Code after changing the setting.

"there is no package called 'shiny'" (or similar)

The package isn't installed. Run install.packages("shiny") in the R terminal. Same logic applies to any missing package — the error message names exactly what's missing.

R Terminal won't start

Double-check the path in R > Rterm: Windows. The file R.exe must actually exist at that location. Open File Explorer and verify by navigating there.

Shiny app launches but is blank or errors

Check the R terminal — error messages from a Shiny app appear there, not in the browser. Common causes: missing package, typo in code, wrong working directory.
